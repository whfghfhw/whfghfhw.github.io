部署原理


使用Hexo需要Node.js环境，假设环境准备完毕，通过命令hexo g(g代表generate)，Hexo会在本地生成源文件，然后通过hexo d(d代表deploy)，发布至远程GitHub，实线箭头部分即可满足使用和配置博客的需要，如果希望进行代码级改造，可使用GitHub Desktop操作.

名称 版本 说明
电脑系统 mac OS X EI Capitan 10.11.6 使用终端操作命令，最好具备linux基础知识
Node.js 10.14.1 Hexo能够运行的基础环境
Hexo 3.8.0 生成静态页面到本地和发布到远程仓库的工具
GitHub Web 远程仓库，存储Hexo发布后的页面
GitHub Desktop 1.5.0 本地仓库，可以拉取远程仓库代码，如果不是代码级深度修改，可不用安装

明白原理就知道为什么要安装这些软件，接下来逐个介绍怎么安装。

安装Node.js

下载后直接安装，下载地址：Node.js

安装Hexo

  • 在Command终端依次执行以下命令

本地Hexo博客文件安装完毕。

  • 启动服务，执行：
    hexo s

  • 浏览器访问：
    http://localhost:4000/
    若出现Hexo默认博客页面，则本地部署成功。

使用GitHub Pages

  • 注册GitHub账号
  • 新建仓库，格式：{你GitHub账户名}.github.io
  • 生成SSH Key，Command执行：
    ssh-keygen -t rsa -C "邮箱地址"
  • 查看并复制公钥内容：
    cat ~/.ssh/id_rsa.pub
  • GitHub页面右上角=>Settings=>SSH and GPG keys=>New SSH key
  • 安装插件,command执行：
    npm install hexo-deployer-git --save
